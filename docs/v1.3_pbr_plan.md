## Vision

Ship a **PBR Material auto-wiring system** as the headline feature of MorfyAI v1.3. Drop texture files, pick renderer engine (Karma or RedShift), MorfyAI auto-detects map types and builds a complete material network.

This becomes MorfyAI's flagship "concrete utility" feature — moving the product beyond commodity AI chat into daily-driver territory for Houdini artists.

**Strategic positioning**:
- Stays inside MorfyAI (NOT spun off as separate "MorfyPBR" product)
- Foundation for future AI material generation (Look Dev Mode, deferred to later release)
- Free tier feature for acquisition; Pro tier upsell = API-based map generation

## Architecture

### Principles
1. Engine independent from UI and AI — single source of truth (`pbr/api.py`)
2. AI is an optional layer on top of the engine — engine works offline
3. Renderer pluggable — adding Arnold/V-Ray later = adding one builder file
4. Detection pluggable — adding pattern presets = adding JSON entries

### Data Model (`pbr/models.py`)

```python
@dataclass
class PBRMap:
    map_type: str              # "basecolor", "normal", "roughness", ...
    file_path: str
    confidence: float          # 0.0–1.0 from detector
    detected_by: str           # "regex" | "fuzzy" | "vision" | "manual"
    colorspace_hint: str       # "sRGB" | "Raw" | "Auto"

@dataclass
class PBRSet:
    name: str                  # "wood_oak"  (derived from common stem)
    maps: dict[str, PBRMap]    # {"basecolor": PBRMap, "normal": PBRMap, ...}
    missing: list[str]         # ["metallic", "ao"]
    udim: bool = False
    source_folder: str = ""

@dataclass
class BuildResult:
    success: bool
    material_path: str         # "/mat/MAT_wood_oak"
    engine: str                # "karma" | "redshift"
    warnings: list[str]
    error: str | None = None
```

### Public API (`pbr/api.py`)

```python
def build_material(
    files: list[str],
    engine: str,                    # "karma" | "redshift"
    target_parent: str = "/mat",
    material_name: str | None = None,
    apply_to: str | None = None,    # node path to assign material
    options: dict | None = None,    # colorspace override, displacement scale, etc.
) -> BuildResult:
    """Detect → group → build → optionally assign. Engine-agnostic entrypoint."""

def detect_set(files: list[str], use_vision: bool = False) -> PBRSet: ...
def list_engines() -> list[str]: ...
def list_supported_maps(engine: str) -> list[str]: ...
def generate_missing_maps(pbr_set: PBRSet, method: str = "analytical") -> PBRSet: ...
```

UI and AI tool both call `build_material()`. Zero code duplication.

## Detection Strategy (Library Random Handling)

Library naming chaos requires multi-tier detection:

### Tier 1: Regex Presets (~80% coverage)
Ship with `presets.json` containing patterns for top libraries:
- **Megascans**: `*_Albedo`, `*_Normal`, `*_Roughness`, `*_Metalness`, `*_AO`, `*_Displacement`
- **PolyHaven**: `*_diff_<res>`, `*_nor_gl`, `*_rough`, `*_arm`
- **AmbientCG**: `*_Color`, `*_NormalGL/DX`, `*_Roughness`, `*_Metalness`, `*_AmbientOcclusion`
- **Poliigon**: `*_COL`, `*_NRM16`, `*_REFL`, `*_GLOSS`
- **Generic**: `basecolor`, `diffuse`, `albedo`, `normal`, `nrm`, `roughness`, `rough`, `_n`, `_r`, etc.

### Tier 2: Fuzzy Match (~15% coverage)
Levenshtein distance + token splitting. Examples:
- `wd_oak_bcol.png` → tokenize → `bcol` ~= `basecolor` (similarity 0.7)
- Confidence score displayed: ✓ (>0.8) / ⚠ (0.5–0.8) / ✗ (<0.5)

### Tier 3: AI Vision Fallback (~5% coverage)
For files where regex+fuzzy fail (confidence < 0.6), use vision LLM:
```
"Look at this thumbnail. Classify as: basecolor / normal / roughness / 
 metallic / ao / height / opacity / emissive / unknown."
```
- Thumbnail 256×256 to minimize token cost
- Cache by file_hash in `cache/pbr_vision_cache.json`
- Skip entirely if regex confidence > 0.9

### Tier 4: Manual Override
UI dropdown per map allows user to override. Save custom regex presets to `config/houdini_ai.ini [pbr_patterns]`.

### UDIM Handling
- Detect patterns: `<UDIM>`, `\.\d{4}\.`, `_<UDIM>_`
- Set `tex.parm("tilingmode").set("udim")` in builders

## What Makes the AI Integration Worth It

**Rule applied**: if a feature works without AI, it does NOT belong in the AI side — put it in the tab. AI integration must use LLM as the *unique enabler*.

### Tier S (AI-only, game-changer — must ship in v1.3)
1. **Vision Map Classifier** — read thumbnail to classify ambiguous filenames (`IMG_4521.jpg`, etc.)
2. **Conversational Refinement** — "make it more wet looking" → adjusts roughness/IOR/specular

### Tier S Deferred (Pro tier candidates, v1.4+)
3. **Auto-Generate Missing Maps via API** — Replicate / Stable Diffusion / Polycam → produce normal/roughness from basecolor only
4. **Reference Image → Material Spec** — drop a photo, AI builds matching material from library

### Tier A (Strong differentiator, v1.4+)
5. **Natural Language Library Search** — "ada texture kayu gelap kaya oak gak?"
6. **Render Error Diagnosis** — screenshot viewport, AI diagnoses flipped normal, wrong colorspace, etc.

### Tier B (Future)
7. **Cross-Renderer Translation** — Karma material → RedShift equivalent
8. **Smart Defaults / Memory** — learn user's preferred displacement scale, common patterns

### Rejected (NOT AI features — pure deterministic)
- Building material network (Python + Houdini API)
- Assigning material to node (`hou.node().setMaterialNode()`)
- Renaming material (string template)
- Detecting filename pattern (regex + fuzzy)
- Calculating UV scale (math)

## Builder Specifications

### Karma MaterialX Builder
- Container: `subnet` inside `matnet`
- Core shader: `mtlxstandard_surface`
- Each texture: `mtlximage` with correct `signature` (color3 / vector3 / float)
- Normal: route through `mtlxnormalmap`
- AO: multiply with basecolor via `mtlxmultiply`
- Displacement: separate output → `mtlxdisplacement`
- Colorspace: OCIO-aware (`srgb_texture` for color, `raw` for data maps)

### RedShift Builder
- Container: `redshift_vopnet`
- Core shader: `redshift::StandardMaterial`
- Each texture: `redshift::TextureSampler`
- Normal: route through `redshift::BumpMap` (input type = "Tangent-Space Normal")
- AO: multiply with basecolor via `redshift::RSColorLayer` (Multiply blend mode)
- Displacement: separate output → `redshift::Displacement` (default scale 0.05)
- Colorspace: string-based (`tex0_colorSpace = "sRGB" | "Raw"`)

### Critical Edge Cases
- **Normal Y-flip**: detect `NormalDX` vs `NormalGL` → set flip_g if DX
- **UDIM**: detect token, set tilingmode
- **OCIO config variation**: handle custom OCIO configs, fall back to standard if not detected
- **Plugin not loaded**: graceful error if RedShift plugin missing
- **Version compat**: test on RS 3.5+, Houdini 19.5+

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| RedShift API changes between versions | High | High | Test on RS 3.5+ LTS. Try/except on node creation. Document supported versions. |
| Karma MaterialX evolution | Medium | Medium | Lock to Houdini 19.5+. Note in README. |
| Vision LLM cost spike | Medium | Medium | Cache per file_hash. Thumbnail-only (256px). Disable by default. |
| Detection accuracy <50% on truly chaotic library | Medium | Medium | Manual override UI + custom regex preset save. Vision fallback. |
| Houdini version compat (19.5 vs 20.x) | Low | High | Test matrix. Feature-flag new APIs. |
| OCIO config variations | Medium | Medium | Detect OCIO env var, use OCIO role names. Document common configs. |
| Build hangs Houdini UI on large batches | Medium | High | Run in background thread with progress. Yield to event loop. |
| AI tool called with bad file paths | Low | Low | Validate paths before invoking builder. |
